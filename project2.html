Project 2 - Configuring a Windows Server 2022 Virtual Machine with Active Directory

Description: This project involved configuring a Windows Server 2022 virtual machine in Oracle VirtualBox, setting up Active Directory Domain Services (AD DS), and promoting the server to a domain controller. The project covers the entire process from installation to basic usage of Active Directory.

Technologies Used: Windows Server 2022, Oracle VirtualBox, Active Directory Domain Services (AD DS), Command Prompt, PowerShell.

Steps to Configure the Virtual Machine:

  1. Setting Up VirtualBox: Created a new virtual machine in Oracle VirtualBox, allocated memory, attached the Windows Server 2022 ISO for installation, and configured system settings including boot order.
  2. Installing Windows Server: Booted the virtual machine from the ISO, followed the installation prompts, chose the appropriate server type (Server with Desktop Experience), and completed the setup of Windows Server 2022.
  3. Configuring Initial Server Settings: Used sconfig to set the computer name, configure the time zone, enable remote management, and set up basic network configuration.
  4. Installing Active Directory Domain Services: Installed AD DS using PowerShell with the command Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools. This installed the necessary features to create a domain.
  5. Promoting the Server to a Domain Controller: Used the Install-ADDSForest command to create a new forest and domain (jamf.local). This process included setting up the Directory Services Restore Mode (DSRM) password.
  6. Configuring Network Settings: Assigned a static IP address to ensure the server could communicate with other machines consistently. Set the IP address, subnet mask, default gateway, and DNS server manually through network settings.
  7. Testing Domain Connectivity: Verified connectivity between the domain controller and a client machine using ping tests. Configured the client to use the server's IP address as the DNS server and attempted to join the domain.
  8. Joining a Client Machine to the Domain: On the client machine, navigated to system settings to change the computer name and domain. Entered the domain name (jamf.local), provided the domain administrator credentials, and successfully joined the client to the domain.

Challenges: One of the main challenges was configuring the network settings properly to ensure that the server and client could communicate. Initially, there were issues with IP address conflicts and DNS configurations, which were resolved by setting a static IP and adjusting the DNS server settings. Another challenge was ensuring that the client machine could properly locate the domain, which required careful DNS configuration.

Solutions: To resolve network configuration issues, I manually set a static IP address on both the server and client machines. I also configured the DNS settings to ensure name resolution was working correctly for the domain. Additionally, I verified firewall settings and ensured both machines were on the same network segment.

Outcome: Successfully set up a Windows Server 2022 virtual machine as a domain controller, configured Active Directory, and connected a client machine to the domain. This project demonstrated my ability to configure a server environment, manage Active Directory for a small network, and troubleshoot network connectivity issues effectively.
